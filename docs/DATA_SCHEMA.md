# Data Schema Reference — Smart-Risk Sentinel

> Authoritative reference for the PostgreSQL schema. The DDL itself lives
> in [`backend/data/schema.sql`](../backend/data/schema.sql); this document
> explains the **intent** of each table and column. If the SQL and this doc
> ever drift, the SQL wins — but raise a PR to bring this doc back in line.

---

## Tables

### 1. `borrowers`
One row per customer. The "who" of the system.

| Column              | Type          | Notes                                                    |
|---------------------|---------------|----------------------------------------------------------|
| `customer_id`       | UUID PK       | Generated by `gen_random_uuid()`                         |
| `name`              | VARCHAR(120)  | Synthetic, from Faker                                    |
| `age`               | INT           | 21–70                                                    |
| `sector_code`       | VARCHAR(20)   | FK → `sector_reference.sector_code`                      |
| `annual_income`     | NUMERIC(12,2) | LKR                                                      |
| `monthly_income`    | NUMERIC(12,2) | Used for DTI; usually `annual_income / 12`               |
| `monthly_obligations`| NUMERIC(12,2)| Sum of existing loan instalments                         |
| `crib_grade`        | VARCHAR(2)    | A, B, C, D, E, XX — maps to CRIB Sri Lanka's 5 risk tiers |
| `net_worth`         | NUMERIC(14,2) | Used for Capital category                                |
| `app_login_freq`    | INT           | Logins per month                                         |
| `province`          | VARCHAR(40)   | Sri Lankan province (Western, Central, etc.)              |
| `created_at`        | TIMESTAMPTZ   | Default `now()`                                          |

### 2. `lease_agreements`
One row per active lease. A borrower can have multiple, but for the
prototype we'll keep it 1:1.

| Column              | Type          | Notes                                                    |
|---------------------|---------------|----------------------------------------------------------|
| `agreement_id`      | UUID PK       | Generated by `gen_random_uuid()`                         |
| `customer_id`       | UUID FK       | → `borrowers.customer_id`                                |
| `vehicle_type`      | VARCHAR(20)   | 'Private' or 'Commercial'                                |
| `vehicle_value`     | NUMERIC(12,2) | Initial purchase price (LKR)                             |
| `loan_amount`       | NUMERIC(12,2) | Principal disbursed                                      |
| `ltv_ratio`         | NUMERIC(5,4)  | `loan_amount / vehicle_value` at origination             |
| `dpd_current`       | INT           | Current days-past-due                                    |
| `dpd_pattern`       | INT[]         | Last 6 months of DPD figures, oldest first               |
| `tenure_months`     | INT           | Lease term: 24, 36, 48, or 60 months                     |
| `start_date`        | DATE          | Lease start                                              |
| `created_at`        | TIMESTAMPTZ   | Default `now()`                                          |

### 3. `market_valuations`
Current resale value of each vehicle — used to recompute the **current**
LTV (vs the origination LTV stored on `lease_agreements`).

| Column              | Type          | Notes                                                    |
|---------------------|---------------|----------------------------------------------------------|
| `valuation_id`      | UUID PK       |                                                          |
| `agreement_id`      | UUID FK       | → `lease_agreements.agreement_id`                        |
| `current_value`     | NUMERIC(12,2) | Latest market resale value                               |
| `valuation_date`    | DATE          |                                                          |
| `liquidity_score`   | INT           | 1 (illiquid) – 10 (highly liquid resale)                 |

### 4. `sector_reference`
Macro / sectoral data used by the Conditions category.

| Column              | Type          | Notes                                                    |
|---------------------|---------------|----------------------------------------------------------|
| `sector_code`       | VARCHAR(20) PK| e.g. 'TRANSPORT', 'TOURISM', 'AGRICULTURE'               |
| `sector_name`       | VARCHAR(100)  |                                                          |
| `npl_ratio`         | NUMERIC(5,4)  | Most recent NPL ratio for the sector (0.0–1.0)           |
| `gdp_outlook`       | VARCHAR(20)   | 'Positive' / 'Neutral' / 'Negative'                      |
| `updated_at`        | TIMESTAMPTZ   |                                                          |

### 5. `risk_scores_log`
Optional audit log of every score the engine has produced. Useful for the
demo to show "the score 30 days ago vs today." Not used by the live
endpoints in v1.0 but the table exists so it can be wired up later.

| Column              | Type          | Notes                                                    |
|---------------------|---------------|----------------------------------------------------------|
| `log_id`            | UUID PK       |                                                          |
| `customer_id`       | UUID FK       | → `borrowers.customer_id`                                |
| `score`             | INT           | 0–1000                                                   |
| `grade`             | VARCHAR(10)   | 'Low' / 'Medium' / 'High'                                |
| `compliance_breach` | BOOLEAN       |                                                          |
| `calculated_at`     | TIMESTAMPTZ   | Default `now()`                                          |

## Relationships

```
sector_reference  (sector_code) ──┐
                                  │
borrowers (customer_id) ──┬───────┘
                          │
                          ├── lease_agreements (agreement_id)
                          │         │
                          │         └── market_valuations
                          │
                          └── risk_scores_log
```

A borrower has 1+ lease agreements (1:1 for v1.0).
Each lease agreement has 1+ market valuations (we keep only the latest).
Each borrower belongs to 1 sector.
Each borrower accumulates risk score log entries over time.

## Conventions

- All primary keys are UUIDs generated by `gen_random_uuid()` from the
  `pgcrypto` extension. The seed script enables this extension.
- All timestamps are `TIMESTAMPTZ` and default to `now()`.
- Money is `NUMERIC(12,2)` for individual amounts, `NUMERIC(14,2)` for
  net worth (which can be larger). Never use `FLOAT` for money.
- LTV is stored as a decimal (`0.50` not `50`) for arithmetic consistency.
- CRIB grades are stored as fixed-width `VARCHAR(2)` to accommodate `'XX'`.
